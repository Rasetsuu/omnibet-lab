# v961-v990 Rust baseline evaluation report

This milestone adds the first real Rust runtime evaluator for the new local data spine.

## Purpose

`omnibet-baseline-eval` reads canonical `matches.jsonl` rows generated by the v891 Football-Data importer and writes:

```text
reports/model_eval.json
```

This is not a trained betting model. It is a deliberately simple baseline used to prove the evaluation/report path and create a minimum benchmark before stronger models are added.

## Baseline v961

The first baseline is:

```text
chronological_expanding_frequency_1x2_v961
```

Behavior:

- sorts matches chronologically
- uses an initial training window
- evaluates on the final chronological slice
- uses expanding historical 1X2 result frequencies with Laplace-style initial counts
- updates only after each evaluated match is scored
- does not use a random split

## Metrics

The report includes:

- accuracy
- log loss
- Brier score
- calibration ECE
- calibration bins

Market scope is explicitly:

```text
1x2 / regulation_90
```

This preserves the phase-aware modeling requirement: regulation markets must stay separate from extra-time, penalties, qualification, and trophy-lift markets.

## CLI

```bash
cargo run --manifest-path rust-core/Cargo.toml --bin omnibet-baseline-eval -- \
  --matches data/local_historical/football_data/normalized/england_premier_league/2024_2025/matches.jsonl \
  --out reports/model_eval.json \
  --min-train 200 \
  --min-eval 50 \
  --eval-fraction 0.20 \
  --source-label football_data_england_premier_league_2024_2025
```

## Expected blocked behavior

If there are not enough rows, the report is still generated but blocked:

```text
ready: false
status: blocked_insufficient_rows
baseline_training_allowed: false
real_model_ready: false
model_status: locked_needs_more_rows_for_eval
```

This prevents fake model readiness.

## Expected completed behavior

Once enough local rows exist, the report can become:

```text
ready: true
status: baseline_eval_complete
baseline_training_allowed: true
real_model_ready: false
model_status: baseline_eval_complete_stronger_model_required
```

The important point is that `real_model_ready` remains false. A simple baseline report is not enough for betting-grade trust.

## Product safety

This milestone makes no profit claim, no staking recommendation, and no betting advice claim.

The purpose is to answer:

```text
Can our future model beat a dumb chronological baseline?
```

not:

```text
Can we bet real money now?
```

The answer to the second remains no until much later evaluation, calibration, CLV, and paper-ledger gates pass.
